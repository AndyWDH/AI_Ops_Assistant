# 客户数据字段说明

## 1. 数据表概览

| 表名 | 记录数 | 字段数 | 描述 |
|------|--------|--------|------|
| customer_base.csv | 10,000 | 12 | 客户基础信息表，包含客户的基本身份、职业、收入等静态信息 |
| customer_behavior_assets.csv | 120,000 | 24 | 客户行为资产表，包含客户的资产分布、产品持有、行为习惯等动态信息 |

## 2. customer_base.csv 字段说明

| 序号 | 字段名 | 数据类型 | 含义说明 |
|------|--------|----------|----------|
| 1 | customer_id | object | 客户唯一标识符，用于关联客户行为资产表 |
| 2 | name | object | 客户姓名 |
| 3 | age | int64 | 客户年龄 |
| 4 | gender | object | 客户性别，可能取值：男、女 |
| 5 | occupation | object | 客户职业，如：教师、工程师、医生等 |
| 6 | occupation_type | object | 职业类型，可能取值：白领、蓝领、自由职业等 |
| 7 | monthly_income | float64 | 客户月收入，单位：元 |
| 8 | open_account_date | object | 开户日期，格式：YYYY-MM-DD |
| 9 | lifecycle_stage | object | 客户生命周期阶段，如：新客户、活跃客户、休眠客户等 |
| 10 | marriage_status | object | 婚姻状况，可能取值：已婚、未婚、离异等 |
| 11 | city_level | object | 城市等级，如：一线城市、二线城市、三线城市等 |
| 12 | branch_name | object | 所属支行名称 |

## 4. 表关系说明

两个表通过 `customer_id` 字段关联，形成一对多关系：
- 一个客户（customer_base表）对应多条行为资产记录（customer_behavior_assets表）
- 每条行为资产记录对应唯一的客户

## 5. 数据更新频率

- customer_base.csv：静态数据，更新频率较低，主要在客户信息变更时更新
- customer_behavior_assets.csv：动态数据，每月更新一次（从stat_month字段可见）

## 6. 数据用途

- **客户画像分析**：通过基础信息和行为资产数据，构建客户360°画像
- **产品推荐**：基于客户的资产分布和行为习惯，推荐合适的金融产品
- **风险评估**：根据客户的收入、资产和行为，评估客户风险承受能力
- **营销触达**：基于客户的活跃程度和冷却期，制定精准的营销计划
- **客户生命周期管理**：根据客户的生命周期阶段，采取不同的运营策略

## 7. 数据质量说明

- 所有字段均已去重，确保数据唯一性
- 客户ID字段无缺失值，确保表间关联完整性
- 数值字段均为有效数值，无异常值
- 日期字段格式统一，便于时间序列分析
